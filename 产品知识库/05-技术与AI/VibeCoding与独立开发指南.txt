【一、什么是Vibe Coding】

定义：
- 我提供vibe，程序员（AI）提供coding
- 完全不写代码，通过自然语言让AI实现想法
- 对文科生来说是"可视化沟通"工具

心态：
- 带着"玩"的心态去vibe coding
- 不需要创业，也可以做产品
- 给自己做个小玩具也是有意义的
- Vibe coding会成为一种自我表达


【二、Vibe Coding的价值】

1. 把想法变成现实
   - 直接跟AI描述想法，它就会帮你变成现实
   - 全程没有写一行代码

2. 与程序员的"可视化沟通"
   - 把产品想法具象化
   - 减少沟通成本
   - 快速验证idea

3. 学AI的最好方式
   - 比看教程更有效
   - 在做的过程中学习
   - Build first, learn later

4. 支配AI牛马的快乐
   - 体验到指挥AI干活的感觉
   - 理解AI的能力边界


【三、工具选择】

主流工具：
- Cursor：AI编程工具，Tab预测功能强
- Windsurf：AI编程工具
- Replit：无代码也能搭建产品，适合新手
- Bolt, Lovable：快速原型工具

如何选择：
- 工具太多，不知道选哪个是正常的
- 建议：看两个对比视频就能理解区别
- 不同工具适合不同场景

Google AI Studio + Gemini：
- 可以用纯自然语言搭建AI小工具
- 直接部署成web app
- 手机上保存到桌面使用


【四、产品Idea怎么找】

1. 产品idea是痛点和技术的连线题
   - 痛点是不变的
   - 技术是一直在变的
   - 关注新技术能解决什么老痛点

2. 做一个小众而有观点的产品
   - 不需要做"全民级产品"
   - 小众但解决真实问题
   - 有个性、有态度

3. 为自己解决问题
   - 自己的痛点最真实
   - 自己是第一个用户
   - AI时代，"只为自己创造"也是一种力量

4. 关注AI技术最新动态
   - 避免重新造轮子
   - 技术在变，要跟上
   - 新能力带来新可能


【五、实际案例】

Zara做过的Side Project：

1. TLDW（Too Long Didn't Watch）
   - 帮助从长视频高效学习
   - 自己的痛点变成产品

2. 自适应语速提词器
   - 用Gemini的实时语音识别
   - 录视频时屏幕显示逐字稿，速度自动匹配

3. AI词典
   - 用Gemini 3 + Google AI Studio
   - 梦中的词典变成现实

4. 学语言App
   - 给纸质书拍照转成可交互的语言学习材料
   - AI讲解

5. 把Twitter feed变成AI播客
   - 实验性项目

6. 把播客改写成纸质书
   - 用Claude把逐字稿改写成书籍形式


【六、开发心态】

1. 不要被满屏代码吓到
   - 代码只是达到目标的手段
   - 重要的是知道要做什么

2. 比技术力更重要的是心态
   - 相信可以做成
   - 遇到问题就问AI
   - 迭代而非完美

3. 担心被抄？想法过时了
   - 是骡子是马拉出来溜溜才知道
   - 刚起步时最大问题是没人用
   - 先做出来再说

4. 人们往往担心成功后才会遇到的问题
   - 还没人用就担心API成本
   - 还没人看就担心被嘲笑
   - 先解决"有人关注"的问题


【七、不足与局限】

文科生不要指望通过vibe coding做出好产品
- Vibe coding是学习和原型工具
- 真正做产品还需要专业程序员
- 吸引程序员加入你，而不是自己转码

正确的定位：
- Vibe coding是"可视化沟通"
- 把想法具象化给程序员看
- 不是替代专业开发


【八、成为Builder的路径】

你不需要创业，也可以成为一个builder：
- 即使在大厂打工
- 即使还在上学
- 都可以用AI工具做side project

可以做的事：
- 用vibe coding做一个AI小工具
- 写一些learn in public的内容
- 在X上分享做产品的过程

Founder ≠ Builder
- 不一定要创业
- 但可以有builder的心态和实践
